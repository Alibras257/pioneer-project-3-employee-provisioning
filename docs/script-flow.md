# Script Flow Documentation

## Overview
This project automates a basic employee onboarding workflow using a Bash script. The script reads employee names from an input file, generates a username for each employee, creates a dedicated employee directory, writes a welcome message, and records operations in a log file.

## Script File
The provisioning script is located at:

```bash
scripts/provision_employees.sh
Workflow Summary
1. Define file and directory paths
At the beginning of the script, the following paths are defined:

INPUT_FILE → path to the employee input list
LOG_FILE → path to the log file
EMPLOYEE_DIR → base directory where employee folders are created
These variables make the script easier to manage and update.

2. Create required directories
The script ensures that the required directories exist before processing begins.

It creates:

logs/
output/employees/
This prevents errors if the folders are missing when the script runs.

3. Logging function
A function named log_message() is used to write messages to the log file.

Purpose
records successful operations
records errors
adds timestamps for traceability
Example log format:

txt
2026-07-06 10:30:00 - SUCCESS: Created directory for John Doe at ./output/employees/john_doe
4. Username generation function
A function named generate_username() is used to standardize employee usernames.

Logic
convert all letters to lowercase
replace spaces with underscores
Example
John Doe becomes john_doe
Ibraheem Aloyinlapa becomes ibraheem_aloyinlapa
This username is used as the employee folder name.

5. Input file validation
Before reading the employee list, the script checks whether the input file exists.

If the file is missing:

an error message is displayed in the terminal
the error is logged
the script exits with a failure status
This prevents the script from running with invalid input.

6. Start of provisioning process
The script:

prints a start message to the terminal
writes a start entry to the log file
This marks the beginning of a provisioning session.

7. Read employee names line by line
The script uses a while loop to read the employee input file one line at a time.

Each line is treated as one employee name.

The loop also ignores empty lines to avoid creating invalid folders.

8. Generate employee directory path
For each employee:

the employee name is read
a username is generated
a path is built for the employee folder inside output/employees/
Example:

bash
output/employees/john_doe
9. Create employee directory
The script creates a directory for each employee using:

bash
mkdir -p
If the directory is created successfully:

a success message is shown in the terminal
the action is written to the log file
If directory creation fails:

an error is shown
the error is logged
the script continues to the next employee
10. Create welcome message file
Inside each employee directory, the script creates a file named:

bash
welcome.txt
The welcome file contains:

employee name
generated username
directory location
onboarding message
This simulates an onboarding communication for the new employee.

11. Log welcome message creation
After creating the welcome file, the script checks whether the operation succeeded.

If successful:

it prints a confirmation message
it writes a success entry to the log file
If unsuccessful:

it prints an error
it writes an error entry to the log file
12. Complete execution
After all employee names have been processed:

the script prints a completion message
the script writes a completion entry to the log file
This indicates the provisioning run has finished.

Script Logic Highlights
The script demonstrates the following Bash concepts:

variables
functions
loops
conditional statements
file handling
directory creation
logging with timestamps
Assumptions
This project is based on the following assumptions:

The script is executed in a Linux environment.
Bash is available on the system.
Employee names are stored one per line in the input file.
The project simulates provisioning by creating employee directories instead of actual Linux user accounts.
The user running the script has permission to create files and folders in the project directory.

Employee folders
The script creates employee directories under:

bash
output/employees/
Welcome files
Each employee folder contains:

bash
welcome.txt
Log file
The script records execution logs in:

bash
logs/provisioning.log
